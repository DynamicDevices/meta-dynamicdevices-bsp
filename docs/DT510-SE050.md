# DT510 — SE050 (same stack as Jaguar Sentai)

## SSOT (hardware doc)

- **SE050** on **I2C4**, **7-bit address `0x48`** — matches the [VIX DT510 pinout](https://docs.google.com/document/d/1dlVcfW7SrOifR-rGjkJnVbnQBO4uU8HeS1MEfDmgcYE/edit?usp=sharing).

## Software stack (already correct for DT510)

DT510 uses the **same SE05x / OpTEE configuration** as **`imx8mm-jaguar-sentai`** (and other Jaguar boards with **`se05x`**):

- **`meta-dynamicdevices-distro`** / **`optee-os-fio`**: **`CFG_CORE_SE05X_I2C_BUS=3`**
- On i.MX8M, **bus index `3`** is the **fourth** I²C controller — i.e. **Linux `&i2c4`**, not `&i2c1`–`&i2c3`.
- The bbappend documents this: *if DeviceTree is e.g. **i2c4**, OpTEE uses the corresponding bus index* (pinmux for SE050 is initialised in OpTEE).

So **Ollie’s “I2C4 @ 0x48”** is **aligned** with the **existing SE050 implementation**, not a different bus.

## Machine configuration

`conf/machine/imx8mm-jaguar-dt510.conf`:

- **`MACHINE_FEATURES`** includes **`se05x`** (when not `LOCAL_DEVELOPMENT_BUILD`)
- **`SE05X_OEFID:imx8mm-jaguar-dt510 = "0xA200"`** — same pattern as Sentai.

## Device tree note

- **Sentai** does not necessarily expose an **SE050 child node** under `&i2c4` in the kernel `.dts`; **OpTEE** talks to SE050 directly using **`CFG_CORE_SE05X_I2C_BUS`**.
- Adding an explicit **`&i2c4`** + **`se050@48`** (or NXP binding) in DT510 is **optional** for kernel visibility / plug-and-trust; it does **not** replace the OpTEE bus configuration.

## Conclusion

The **DT510 SE050 placement** (I2C4, `0x48`) is **consistent** with the **existing factory SE05x / OpTEE** setup used on Sentai. Further work is mainly **optional DT completeness** and **lab validation**, not a different security architecture.
